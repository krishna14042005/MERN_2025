Why Use Capital Letters for Components?

In JavaScript, specifically in frameworks like React, it's a convention to use capital letters when defining components. This helps distinguish components from regular HTML elements.

- Component Naming Convention: In JSX (JavaScript XML), the convention is that custom components must start with a capital letter. For example, MyComponent. This helps React differentiate between built-in HTML elements and your custom components. For instance, <div> is a regular HTML element, whereas <MyComponent> is a custom component.

- Why Capitalize? If you use lowercase letters for components (e.g., <myComponent>), React would treat it as a string, resulting in it being interpreted as a regular DOM tag, which would lead to incorrect behavior.

Why Use Default Export?

In JavaScript ES6, the default keyword allows you to export a single entity from a module, making it the default export.

- What is Default Export? When you use export default, it means you are exporting one primary item from the module. For instance, in a React component, you often export the component itself as the default export. 

export default MyComponent;

- Why Use Default Export? The default export allows you to import the module without needing to know the exact name of the entity being exported. This makes the import syntax more flexible and cleaner. 

import MyComponent from './MyComponent';

This syntax simplifies the import because you don't need to match the exported name with the import statement.
